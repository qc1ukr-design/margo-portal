# PROJECT_GOAL.md — Margo Portal

## Мета проєкту
Клієнтський портал бухгалтерської компанії "Марго" (SaaS, multi-tenant).
Модуль звірки чеків: автоматично зіставляє фіскальні чеки з банківськими надходженнями через ланцюжок API.

## Визначення успіху
- Нуль ручних звірок для клієнтів сценаріїв А–Е
- Автоматичне виявлення розбіжностей і помилок (MATCHED_CROSS_ENTITY, delta_amount поза нормою)
- Мульти-тенантна ізоляція даних (RLS на всіх таблицях)
- Час роботи алгоритму звірки: < 5 хв на клієнта

## Scope
**В scope:** Checkbox, NovaPay (agent + bank), ПриватБанк, Nova Poshta, Вчасно, Monobank, LiqPay, Prom.UA, Rozetka, Casta, ДПС Електронний кабінет, Cashalot
**Поза scope:** ручний імпорт xlsx у production (виняток: email-реєстри маркетплейсів — автообробка)

## Команда
Один розробник: власник продукту + Claude Code.
Velocity: 10-12 SP/sprint (2 тижні).

## Поточний статус
Stage 0 — Discovery (активний)
